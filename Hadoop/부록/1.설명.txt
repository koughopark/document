빅데이터 분석 도구는 분산 저장소(하둡)의 활용 범위에 따라 직접 분석 도구와 간접 분석 도구로 나눌 수 있다.

# 간접분석도구      하둡      직접 분석 도구
	R               하이브         하이브
	텐서플로        스파크         임팔라
	카페              피그          피그
	테아노            ...          스파크
	....                            ...

- 직접 분석 도구는 하이브, 스파크, 일팔라, 피그 등이 있다.
- 직접 분석 도구는 하둡 파일시스템에 직접 접근하면서 분석 명령(쿼리, 알고리즘)이 하둡의 데이터 노드 상에서 분산 실행되고 분석 결과만 돌려받는다.
- 이 같은 환경을 스케일 아웃 방식의 분산 분석 환경이라고도 하며, 대규모 데이터(수기가~테라바이트 이상)를 분석할 때 적합하다.
- 반면, 간접 분석 도구는 하둡에 적재된 데이터에 직접 접근하는 것은 가능하지만 분석 작업을 하둡에서 실행할 수는 없고 분석 할 데이터를 분석 도구의 컨텍스트로 가져와 실행하는 방식이다.
- 간접 분석 도구로는 R, 텐서플로(Tensorflow), 웨카(Weka), 카페(Caffe), 테아노(Theano) 등이 있다.
- 간접 분석 환경을 스케일업 방식의 독립형 분석 환경이라고도 하며, 소규모 데이터셋을 대상으로 다양한 분석 라이브러리와 시각화로 데이터를 탐색한 뒤
고속의 병렬처리 (GPGPU, General-Purpose Computation on Graphics Hardware)를 수행할 때 사용한다.

- 부록에서는 빅데이터 간접 분석 도구를 이용해 좀 더 다양한 스마트카 빅데이터 분석을 진행한다.
- 사용할 간접 분석 도구는 R, 텐서플로다.

- 부록 A: R을 이용한 회귀분석 - 스마트카 배기량에 따른 운전자의 연소득 예측
- 부록 B: 텐서플로 이용한 신경망 분석 - 주행 중 스마트카의 사고 위험 징후 판별
