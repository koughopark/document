빅데이터 탐색 단계가 데이터를 관찰하고 이해하는 과정이라면
빅데이터 분석은 탐색과 분석을 반복하며 의미 있는 데이터를 추출해 문제를 명확히 정의하고 해결하는 과정이다.

분석의 목적에 따라 크게 5가지 분석 유형이 사용된다.
- 기술 분석: 분석 초기 데이터의 특징을 파악하기 위해 선택, 집계, 요약 등 양적 기술 분석을 수행
- 탐색 분석: 업무 도메인 지식을 기반으로 대규모 데이터셋의 상관관계나 연관성을 파악
- 추론 분석: 전통적인 통계분석 기법으로 문제에 대한 가설을 세우고 샘플링을 통해 가설을 검증
- 인과 분석: 문제 해결을 위한 원인과 결과 변수를 도출하고 변수의 영향도를 분석
- 예측 분석: 대규모 과거 데이터를 학습해 예측 모형을 만들고, 최근의 데이터로 미래를 예측

이러한 다양한 분석 기술을 통해 빅데이터의 가치는 "Raw 데이터 -> 정보 -> 통찰력-> 가치" 순으로 변하게 된다.
통찰력을 갖게 되는 단계에서 빅데이터의 활용 효익이 만들어 지는데, 상품/서비스 개발, 마케팅/캠페인 지원, 리스크 관리 업무 등의 의사결정에
빅데이터를 활용하면서 비즈니스 가치를 본격적으로 창출한다.

빅데이터 분석이 전통적인 RDBMS기반 분석과 다른점은
	첫째, 규모와 성능
	둘째, 외부 데이터와의 결합

고객 라이프스테이지 분석
	라이프스테이지 분석은 사람의 라이프사이클을 단계별로 정의해, 각 단계별로 전략을 달리 수립해 고객을 효과적으로 관리하는 것이다.


1. 임팔라 소개
	실시간 빅데이터 분석 질의가 가능
	하둡의 분산 노드에서 대규모 실시간 분석을 위해 Impalad, Statestored, Catalogd라는 컴포넌트가 설치된다.
		Impalad: 하둡의 데이터노드에 설치되어 임팔라의 실행 쿼리에 대한 계획, 스케줄링, 엔진을 관리하는 코어영역
		Statestored: 분산 환경에 설치돼 있는 Impalad의 설정 정보 및 서비스를 관리
		Catalogd: 임팔라에서 실행된 작업 이력들을 관리하며, 필요 시 작업 이력을 제공
	임팔라를 활용해 하이브 쿼리를 임팔라 쿼리로 바꾸고, 스마트카 데이터셋을 실시간 탐색
	하이브 쿼리는 대부분 임팔라 쿼리와 호환 가능하며, 하이브 대비 빠른 응답 속도를 보장

2. 제플린 소개
	스파크를 기반
	분석 및 시각화 툴
	제플린을 이용하면 웹 UI의 NoteBook에서 스파크 또는 스파크 SQL을 작성해 하둡 클러스터에 작업을 요청하고,	처리결과를 다시 웹 UI에서 시각화해서 볼 수 있다.
	마트 데이터를 대상으로 제플린에서 스파크 SQL을 이용해 다양한 애드혹 분석을 수행
	스파크-SQL을 이용해 영역별 마트 데이터에 접근해 데이터를 추출, 처리, 분석하고 그 결과를 제플린의 시각화 기능을 이용해 다양하게 시각화한다.

3. 머하웃 소개
	머하웃(Mahout)은 하둡 생태계에서 머신러닝 기법을 이용해 데이터 마이닝을 수행하는 툴
	하둡의 분산 아키텍처를 바탕으로 텍스트 마이닝, 군집, 분류 등과 같은 머신러닝 기반 기술이 내재화
	하둡의 분산 환경 위에 맵리듀스를 기반으로 고급 분석을 지원하는 라이브러리 패키지이다.
	머하웃의 라이브러리를 활용해 스마트카의 Managed 영역에 적재된 "스마트카 상태 데이터"와 "운전자운행 데이터"를 이용해 감독학습-분류, 비감독학습-군집의 머신러닝을 수행한다.
	추천기능을 활용해 "차량용품 구매 이력 데이터"를 분석하고 스마트카 운전자 가운데 유사 그룹간의 구매 선호도에 따라 차량용품을 추천하는 작업을 한다.

4. 스쿱 소개
	RDBMS와 HDFS 사이에서 데이터를 편리하게 임포트하거나 익스포트해주는 소프트웨어가 스쿱(Sqoop)이다.
	스쿱은 두가지 버전이 있다
		스쿱 1: CLI 기반으로 스쿱 명령을 실행하는 클라이언트 버전
		스쿱 2: 스쿱서버를 두고 스쿱 클라이언트가 API를 호출하는 방식으로 스쿱 1을 확장한 서버 버전
	스쿱1 아키텍처: 스쿱의 CLI로 임포트, 익스포트 명령을 하둡에 전달하면 맵 태스크가 병렬로 실행되어 외부 데이터베이스와 HDFS 사이에서 대량의 데이터를 임포트 및 익스포트 할수 있는 아키텍처를 제공
	스쿱2 아키텍처: 스쿱 1에서 클라이언트마다 설치됐던 커넥터와 라이브러리를 스쿱 서버에 배치하고 스쿱의 임포트, 익스포트 기능을 REST API로 제공해서 클라이언트를 경량화했다. 스쿱의 주요 기능을 중앙 집중화함으로써 잡을 통합 관리하고 스쿱 1에서 할 수 없었던 접근 통제 가능
	하이브, 임팔라, 제플린, 머하웃 등에서 분석된 결과를 외부 RDBMS 시스템에 편리하게 제공
